Blank Page Troubleshooting

Solving the white screen of death in PHP applications

Why PHP Shows a Blank Page

A blank page in PHP typically means there's a fatal error that's preventing the page from rendering, but error reporting is turned off. Common causes include:

  • Syntax errors in PHP code
  • Missing files or includes
  • Fatal errors that halt script execution
  • Incorrect file permissions
  • Server configuration issues

Step-by-Step Troubleshooting

Enable Error Reporting

Add these lines at the top of your PHP file to display errors:


ini_set('display_errors', 1);
ini_set('display_startup_errors', 1);
error_reporting(E_ALL);
?>

Alternatively, you can add this to your .htaccess file:

php_flag display_errors on
php_value error_reporting -1
Check for Syntax Errors

Use the command line to check your PHP files for syntax errors:

php -l yourfile.php

Common syntax errors include:

  • Missing semicolons
  • Unclosed brackets, parentheses, or quotes
  • Missing PHP opening or closing tags
Verify File Includes

Blank pages often occur when included files are missing or have errors:

// Check if file exists before including
if (file_exists('config.php')) {
    require_once 'config.php';
} else {
    die('Config file not found');
}

Make sure all file paths are correct and files exist.

Check Server Error Logs

Server error logs often contain details about what's causing the blank page:

# Apache error log location (typical paths)
/var/log/apache2/error.log
/var/log/httpd/error_log
/usr/local/apache/logs/error_log

If you don't have server access, create a PHP file with:


phpinfo();
?>

And look for the "error_log" directive.

Quick Checklist

  • Enable error reporting in PHP
  • Check for syntax errors with php -l
  • Verify all included files exist
  • Check file permissions (644 for files, 755 for directories)
  • Look for fatal errors in server logs
  • Check for infinite loops or memory exhaustion
  • Verify PHP is not running out of memory
  • Test with a minimal PHP file ('); ?>)
